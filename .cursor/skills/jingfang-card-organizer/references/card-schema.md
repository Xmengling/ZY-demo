# 方剂卡片字段与整理模板

## 卡片模块

学习笔记版内容使用以下顺序：

```markdown
## 方剂名
### 组成：
### 相关条文
### 方剂名病理
### 辩证要点
### 临床症状：
### 现代疾病：
### 胡希恕
讲稿原文：
要点总结：
### 李冠杰
讲稿原文：
要点总结：
### 对比
几个以……为主的方子对比：A、B、C
1. ...
2. ...
### 医案
医案1：...
十二字病理分析：
...
```

## JSON 载荷字段

网页需要类似下面这样的 JSON 载荷：

```json
{
  "id": "da-xian-xiong-tang",
  "name": "大陷胸汤",
  "categories": ["里证", "半证", "水证"],
  "composition": "大黄6 芒硝16 甘遂1",
  "herbImages": ["大黄.jpg", "芒硝.jpg", "甘遂.jpg"],
  "pathology": [
    {"label": "里热", "text": "[[**舌上燥而渴**]]，热结在里。", "category": "里证"}
  ],
  "pathologySymptoms": ["里热：[[**舌上燥而渴**]]，热结在里。"],
  "mainSymptoms": ["[[**心下痛，按之石硬**]]"],
  "clinicalSymptoms": ["[[**心下痛，按之石硬**]]", "便秘"],
  "modernDiseases": ["急性腹膜炎"],
  "abdominalDiagnosis": "腹诊要点。",
  "huXishuAnalysis": "讲稿原文：...\n要点总结：...",
  "liGuanjieAnalysis": "讲稿原文：...\n要点总结：...",
  "diagnosisPoints": ["[[**核心辨证抓手**]]。"],
  "classicTexts": ["《伤寒论》135：..."],
  "caseItems": ["医案1：...\n十二字病理分析：..."],
  "cases": "医案1：...\n十二字病理分析：...",
  "caution": "禁忌和边界。"
}
```

## 字段说明

- `categories`：从 `表证`、`里证`、`半证`、`水证`、`血证`、`气证`、`阴证` 中选择。
- `composition`：只写中药和剂量，不写煎服法、服用法、先煎后下。
- `herbImages`：知道图片文件名时填写；前端也会从组成中自动匹配。
- `pathology`：每项包含 `label`、`text`、`category`。
- `pathologySymptoms`：给搜索和卡片使用的病理行，格式如 `里热：...`。
- `mainSymptoms`：最核心、最高频、最该背的主症短句。
- `clinicalSymptoms`：完整症状列表，尽量覆盖主症、兼症、舌脉、寒热、饮食、二便、胸腹咽喉等。
- `modernDiseases`：可对应的现代病名或临床场景；不可把现代病名等同于方证。
- `abdominalDiagnosis`：腹诊或触诊要点。
- `classicTexts`：一条文一项，尽量保留编号或出处；关键原文短语用 `[[**...**]]`。
- `diagnosisPoints`：辨证抓手、适用边界、禁忌、类方鉴别点。
- `huXishuAnalysis` / `liGuanjieAnalysis`：分别包含 `讲稿原文：` 和 `要点总结：`。
- `caseItems`：每则医案包含患者信息、主诉/症状、舌脉、处方、疗效，并加 `十二字病理分析：`。
- `cases`：等于 `caseItems` 用两个换行拼接后的完整文本。
- `caution`：禁忌、误用风险、急重症提醒等。

## 标红规则

用 `[[**...**]]` 实现卡片中的红色加粗。优先标：

- `相关条文` 中的主症、兼症、禁忌症状、鉴别短语。
- `病理` 中的证据和核心病机。
- `辩证要点` 中的主抓手、适用边界、禁忌。
- `临床症状` 中的主症、舌脉和关键兼症。
- `医案` 中的主诉、舌脉、疗效和十二字病理证据。
- `对比` 中最容易混淆的鉴别点。

不要把每句话都标红；只标学习时最该抓住的部分。
